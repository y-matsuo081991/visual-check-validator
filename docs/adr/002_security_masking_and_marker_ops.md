# ADR-002: エッジマスキングによる機密保護とマーカー運用の限定化

## Status
* Accepted

## Context (背景と課題)
データセンターの配線監査において、以下の2つの致命的な「技術と運用の壁」が存在した。
1. **セキュリティの壁:** サーバーラックには、IPアドレスやホスト名、資産管理番号など、社外流出が許されない「超・機密情報」が多数貼られている。これを証拠画像としてそのままGCP（クラウド）に同期すると、重大なセキュリティ違反（インシデント）となる。
2. **AI認識精度と運用コストの壁:** 細い光ケーブルや小さなポート番号を画像だけで100%正確に読み取ることは難しい。解決策として「QRコードやARマーカー」を貼るアプローチがあるが、数万本の全ケーブルに貼る運用は人件費（運用コスト）の観点で破綻する。

## Decision (決定事項)
1. **Defensive Masking (エッジ側マスキング):** 画像をGCPへアップロードする前に、必ずエッジ（タブレット側）のプロセスで「ARマーカーの周辺（対象ポート）以外を黒塗り（マスキング）する」あるいは「OCRで検知したテキストをぼかす」前処理を実装する。クラウドには**マスキング済みの安全な画像のみ**を送信する。
2. **マーカー運用の対象絞り込み (ROI最適化):** 全ポートへのマーカー貼付は行わず、ビジネスインパクトの観点から「間違えると致命的な障害に直結するコアスイッチのアップリンク（全体の約5%）」等の重要箇所にのみマーカー運用を限定する。

## Consequences (結果・影響)
* **[Good] セキュリティ監査の確実な突破:** 機密情報が物理的にクラウドへ流出しないアーキテクチャ（データ・サニタイズ）とすることで、情報システム部門のセキュリティ監査をスムーズに通過できる。
* **[Good] 運用コストの適正化:** マーカー運用を「絶対にミスが許されない箇所」に限定することで、現場の作業負担（Chesterton's Fence）を最小限に抑えつつ、システムのROIを最大化できる。
* **[Bad/Trade-off] エッジ端末への負荷:** AIの推論に加え、画像加工（マスキング処理）も端末側で行うため、タブレットのバッテリー消費やCPU負荷が高くなる。

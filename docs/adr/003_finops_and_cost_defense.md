# ADR-003: クラウドコスト防衛とFinOps/MLOps戦略の導入

## Status
* Accepted

## Context (背景と課題)
本システム（VCV）をPoCから全社へスケールアウトさせる際、クラウドのランニングコスト（OpEx）の肥大化が最も警戒される。
特にGCPのVertex AI等を利用してモデルを配備した場合、「リクエストが0件でもエンドポイントの維持費（待機時間）による常時課金が発生する（Scale-to-Zero非対応）」というクラウド特有の罠があり、PoCのROIを著しく悪化させるリスクがあった。
一方で、コスト削減のために監査画像をすぐ削除したり、クラウドへ一切データを送らない設計にすると、エンタープライズに求められる「監査証跡の保存義務」違反や「AIの劣化（Model Drift）の放置」という致命的な運用リスクを生むことが判明した。

## Decision (決定事項)
コスト最小化と、コンプライアンス・品質保証を両立するため、以下のアーキテクチャを導入する。

1. **AI学習・推論コストの完全ゼロ化 (Offline Training & Edge AI):**
   高額な待機コストが発生するクラウドAIの配備を避け、PoC段階のAIモデル学習はTeachable MachineやGoogle Colab等の完全無料環境へオフロードしてTensorFlow.jsモデルを生成する。推論も端末側（Edge AI）で実行することで、AI関連コストを恒久的にゼロ化する。
2. **監査証跡の長期保存とコスト最適化 (Storage Tiering):**
   Cloud Storageの画像に対し「削除」ではなく、「作成から30日経過したファイルは、1GBあたり月額数円の『Archiveクラス』へ自動階層化（Tiering）する」ライフサイクル管理を適用する。これにより、監査に必要な数年間の証跡保存義務を低コストで満たす。
3. **AI品質の継続的監視 (Data Feedback Loop):**
   Edge AIの判定確信度（Confidence Score）が低い「スレスレの画像（全体の数%）」のみをサンプリングし、マスキングせずに再学習・監視用のセキュアバケットへ同期するMLOps設計を組み込む。

## Consequences (結果・影響)
* **[Good] PoCの低ハードル化と監査対応:** GCPの「Always Free」枠にほぼ収まるコスト感でありながら、大企業の厳しいセキュリティ・監査要件（証跡保存）をクリアしているため、経営層・情シス双方から承認を得やすい。
* **[Good] AI品質の持続性:** サンプリングデータをクラウドに返すことで、現場の照明環境の変化などによるAIの劣化（Model Drift）を早期に検知し、モデルの再学習ループ（MLOps）を回すことができる。
* **[Bad/Trade-off] エッジとクラウドのロジック複雑化:** 「確信度が低い場合は生データを送る、それ以外はマスキングする」といった条件分岐がエッジ側に追加されるため、フロントエンドの実装がやや複雑になる。

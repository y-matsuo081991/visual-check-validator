# ADR-001: オフライン対応のEdge AIとSync-Awareな同期設計の採用

## Status
* Accepted

## Context (背景と課題)
データセンターのサーバールーム内は厳密なセキュリティ壁に囲まれた「完全な圏外（電波暗室）」であることが多い。
初期案の「画像をクラウド（GCP）にアップロードし、Cloud Vision APIで判定する」アーキテクチャでは、作業者が電波のある場所（建物の外）へ移動するまで判定結果（OK/NG）が分からず、NGだった場合に再度入室・やり直しが発生するという致命的なUXの欠陥（業務停止）があった。
また、オフラインで動作させた場合、事後のバックグラウンド同期が失敗（GCP側のエラーやトークン切れ等）すると、エビデンスがクラウドに届かないまま「作業完了」と見なされる「Silent Failure（静かなる失敗）」のリスクがあった。

## Decision (決定事項)
1. **Edge AI (TensorFlow.js) の採用:** クラウドAPIへの依存を完全に棄却し、会社貸与のタブレット（専用端末）のローカル環境内で軽量な推論モデルを実行し、0.1秒でOK判定を返すアーキテクチャとする。
2. **Sync-Aware UX (同期完了を以てDoDとする設計):** 端末側でOKが出ても「作業完了」とはせず、UI上に「未同期〇件」というステータスバッジを強制表示する。通信エリアへ復帰後、GCPへ自動同期を行い、確実に `200 OK` が返却されて未同期が0件になった時点を「最終的な作業完了（退室可能）」と定義する。
   * **【技術的補足: iOS Safariの制約回避】** 現在のiOS（iPhone）はプライバシー制約により `Background Sync API` をサポートしておらず、アプリを閉じた状態（裏側）での自動通信が不可能である。本システムではこのプラットフォーム制約を吸収するため、作業者に「アプリ画面（フォアグラウンド）を開いたまま未同期バッジが消えるのを見届ける」ことを運用ルールとして強制し、iOS環境でも確実なエビデンス送信を担保する設計としている。

## Consequences (結果・影響)
* **[Good] 業務レイテンシの極小化:** 外部のIoTデバイス（センサー）による判定と同様に、クラウドの応答を待たず現場（エッジ）で即座に判定が出るため、現場作業のスピードを一切阻害しない。
* **[Good] エビデンス消失（Silent Failure）の防止:** バックグラウンド同期とUXを連携させることで、クラウド側の障害や通信断によるデータ消失を運用で確実に防ぐことができる。
* **[Bad/Trade-off] AIモデルのライフサイクル管理 (OTAアップデート):** クラウドAPIと異なり、AIモデルが端末内に存在するため、新しい機材が導入されてAIモデルを更新する際、全タブレットへモデル（重みファイル）を配布・更新する仕組み（Over-The-Airアップデート等）を別途設計する必要があり、保守コストが増大する。
